# 功能规格说明书：精简作品元数据与布局

**功能分支**: `001-streamline-content-hierarchy`  
**创建日期**: 2026-04-18  
**状态**: 草案  
**输入**: 用户描述: "页面上可显示的 item 多一点。现在一行只有三个有些浪费"；"继续完善，文字内容占比还是太大了，去除作品标题字段，投稿处也不需要这个字段"

## 用户场景与测试 *(必填)*

### 用户故事 1 - 更紧凑的内容浏览 (优先级: P1)

作为 Prompt Gallery 的访客，我希望在首页一次看到更多作品，并且每张卡片让视觉内容优先于文字，这样我能更快完成浏览，而不是被过多说明文字打断。

**为什么选择此优先级**: 这是当前最直接的体验问题，影响首页浏览效率，也决定整个画廊是否显得轻快。

**独立测试**: 打开画廊页面，对比调整前后，在标准桌面屏幕上验证每行作品数量明显增加，且任意卡片的视觉内容区域明显大于文字说明区域。

**验收场景**:

1. **假设** 使用标准桌面屏幕查看首页, **当** 我浏览画廊网格时, **那么** 我应能在同一行看到多于 3 个作品卡片。
2. **假设** 我快速扫视任意一张作品卡片, **当** 我比较图片区域和文字区域时, **那么** 我首先感知到的是作品内容而不是大片文字。
3. **假设** 使用平板或窄屏设备查看首页, **当** 网格自动调整时, **那么** 页面仍应保持较高浏览效率且不出现文字拥挤或换行失控。
4. **假设** 我快速浏览一排作品卡片, **当** 我扫视卡片底部的标签区域时, **那么** 我应能比其他辅助标签更快识别出模型标签。

---

### 用户故事 2 - 去除作品标题字段 (优先级: P2)

作为浏览作品和准备投稿的用户，我希望作品不再带有单独的标题字段，这样界面可以减少一层文字信息，注意力集中在画面、提示词和必要元数据上。

**为什么选择此优先级**: 用户明确指出不需要作品标题字段。去掉这个字段后，卡片、详情和投稿表单都会显著减字，才能真正解决“文字内容占比太大”的问题。

**独立测试**: 分别检查首页卡片、详情弹层和投稿弹层，确认作品标题不再显示，也不再要求填写作品标题。

**验收场景**:

1. **假设** 我浏览画廊首页, **当** 我查看任意作品卡片时, **那么** 卡片不应显示作品标题字段。
2. **假设** 我打开作品详情弹层, **当** 我查看作品信息时, **那么** 详情中不应显示作品标题字段。
3. **假设** 我打开投稿区域, **当** 我填写投稿表单时, **那么** 表单不应要求填写作品标题字段。
### 边缘情况

- 当作品缺少标题后，系统仍需让用户通过预览图、提示词片段、标签或模型信息区分不同作品。
- 当作品描述较长时，文字信息仍需保持可读，但不能重新挤压主视觉区域。
- 当旧数据中仍含有标题字段时，界面应忽略它，而不是继续展示。
- 当卡片同时展示多个标签时，模型标签仍需保持更高辨识度，不能与其他次级标签混在一起。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 画廊列表必须提高单屏可见作品数量，在标准桌面宽度下每行展示数量必须大于当前的 3 个。
- **FR-002**: 作品卡片必须降低文字内容在卡片中的视觉占比，使作品预览内容成为主视觉焦点。
- **FR-003**: 所有面向用户的作品展示界面必须移除作品 `title` 字段。
- **FR-004**: 投稿表单必须移除作品 `title` 输入字段，同时保留清晰的表单上下文。
- **FR-005**: 详情区域必须保留必要内容与操作，但不得依赖作品标题字段作为主要识别信息。
- **FR-006**: 调整后的页面在移动端、平板和桌面端都必须保持信息层级清晰，不得因压缩文字而导致可读性明显下降。
- **FR-007**: 作品卡片中的模型标签必须通过更明显的视觉样式与其他辅助标签区分开，例如使用更高对比度的颜色表达。

### 关键实体 *(如果功能涉及数据，请包含此部分)*

- **GalleryItem**: 代表一个作品条目，包含作品预览、简要说明和用户可见元数据，不再依赖 `title` 字段。
- **ContributionEntryPoint**: 代表用户进入投稿流程的页面区域或弹层入口，包含操作提示和提交动作。

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 在标准桌面屏幕下，首页画廊每行可见作品数量从 3 个提升到至少 4 个。
- **SC-002**: 在首页、详情区和投稿区中，作品标题字段的可见性和可填写性均为 0。
- **SC-003**: 在常见移动端和桌面端视口下，用户都能在 5 秒内识别作品内容与投稿入口，不因作品标题字段移除产生理解障碍。
- **SC-004**: 在首页卡片同时存在多个标签时，用户能够一眼区分模型标签与其他辅助标签，而不需要先打开详情。

## 假设

- 投稿区域在去除作品标题输入后，仍可以通过字段标签、按钮文案和上下文位置让用户理解其用途。
- 用户更倾向于通过更大的内容密度和更轻的文案层级来完成浏览，而不是通过额外说明文字理解页面。
