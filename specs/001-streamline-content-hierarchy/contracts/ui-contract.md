# UI Contract：精简作品元数据与布局

## 首页 (`/`)

- 页面进入后，用户应优先看到导航、筛选控件和作品网格。
- 作品网格在标准桌面宽度下必须每行显示至少 4 个作品卡片。

## 作品卡片

- 每张卡片必须保留作品预览图/视频和必要的辅助元数据。
- 每张卡片不得显示作品标题字段。
- 每张卡片不得新增长段描述文字。
- 模型标签必须比其他辅助标签更醒目，可通过更高对比度的颜色或强调样式突出。
- 卡片交互 `hover`、打开详情、快速复制必须保持可用。

## 详情弹层

- 详情弹层不得显示作品标题字段。
- 下架申请入口必须继续可用。

## 投稿弹层

- 投稿弹层不得要求填写作品标题字段。
- 上传方式切换、字段标签、关闭按钮和提交按钮必须足以表达当前上下文。

## 回归要求

- 搜索、分类筛选、详情打开、快速复制、投稿提交入口不能因标题移除而失效。
- 页面在移动端与桌面端均需保持信息层级清晰。
