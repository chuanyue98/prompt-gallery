# 研究报告：精简画廊信息层级

## 决策 1：继续提高首页密度，但优先压缩文案层级而不是单纯继续塞列数

- **Decision**: 保持当前“多列网格”的方向，同时进一步减少首页 hero 和卡片文字占比，让密度提升更多来自文案缩减与间距优化。
- **Rationale**: 现有网格已经达到 `lg:4 / xl:5 / 2xl:6` 的高密度状态，用户继续反馈“文字内容占比太大”，说明剩余问题主要在信息层级而不是列数不足。
- **Alternatives considered**:
  - 继续提高到更激进的列数：会明显压缩卡片可读性，且容易破坏 hover 预览和点击区域。
  - 保持首页标题与说明文案不动，只收紧卡片：无法解决首屏文案占比过高的问题。

## 决策 2：首页 hero 去标题化，保留氛围背景与导航

- **Decision**: 移除首页显著的 hero 标题与说明文案，把首屏重点让给筛选区和作品网格。
- **Rationale**: 当前 `page.tsx` 里的大标题、badge 和双行描述占用大量首屏垂直空间，直接稀释了画廊本身的展示密度。
- **Alternatives considered**:
  - 缩小 hero 标题字号：仍保留重复信息层级，只是噪音更小但没有消失。
  - 只删说明文案保留主标题：仍会留下较强的视觉阻断，达不到“去除标题”的诉求。

## 决策 3：详情弹层保留作品标题，移除说明性区块标题

- **Decision**: 详情弹层保留作品标题本身，但移除如 `Perspective`、`Syntax` 这类说明性标签，改用更轻的排版表达内容分区。
- **Rationale**: 作品标题属于内容标识，不应删除；真正冗余的是帮助用户“读懂布局”的辅助标签，它们在上下文已经足够明确时会放大文字噪音。
- **Alternatives considered**:
  - 连作品标题一起去掉：会让详情内容缺乏识别点，不符合 spec 里的边缘情况要求。
  - 保留所有标签但缩小字号：只是弱化，而不是去除冗余层级。

## 决策 4：投稿区域取消显式标题，通过上下文与字段文案保持可理解性

- **Decision**: 移除 `ContributeModal` 顶部“我要投稿”标题，改由关闭按钮、字段标签、提交按钮和上传/链接模式切换共同承担上下文提示。
- **Rationale**: 投稿弹层已经具有足够强的场景线索；额外的大标题只会增加顶部文字堆叠。
- **Alternatives considered**:
  - 只把标题缩成副标题：仍占据顶部空间，不能实质减少文字层级。
  - 完全去掉所有字段标签：会牺牲表单可理解性，风险过高。
