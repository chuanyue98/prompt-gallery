# 数据模型：精简作品元数据与布局

## GalleryItem

- **说明**: 首页卡片和详情弹层展示的核心作品实体。
- **关键字段**:
  - `slug`: 唯一标识，用于列表 key、详情选择和删除申请。
  - `description`: 作品简述，在详情中保留正文价值，但需要降低辅助标签干扰。
  - `content`: 提示词正文，用于复制和详情展示。
  - `tags`: 标签集合，用于筛选与卡片底部辅助信息。
  - `model`: 模型名称，作为轻量元数据保留。
  - `media` / `mediaUrl` / `mediaPath`: 作品封面与预览媒体来源。
- **展示规则**:
  - 卡片以媒体内容为主，标签和模型信息为辅。
  - 详情中展示描述、提示词和操作，但不展示作品标题字段。

## GallerySurface

- **说明**: 首页用于承载筛选器、网格和空状态的展示区域。
- **组成部分**:
  - 搜索输入
  - 分类筛选按钮
  - 作品网格
  - 加载失败/无结果状态
- **约束**:
  - 首屏必须让作品网格尽快进入视野。
  - 不再依赖作品标题字段作为卡片主文案。

## ContributionEntryPoint

- **说明**: 用户进入投稿流程的交互入口与弹层上下文。
- **组成部分**:
  - 上传文件 / `Media URL` 模式切换
  - 必填字段与可选字段
  - 提交按钮
  - 关闭按钮
- **约束**:
  - 去除作品标题输入后，仍需保持表单用途清晰。
  - 字段标签和按钮文案必须足以解释当前流程。

## DetailPresentation

- **说明**: 用户查看单个作品时的弹层展示状态。
- **状态**:
  - `default`: 查看作品媒体、描述、提示词和来源。
  - `take-down-request`: 展开下架申请输入区。
  - `take-down-success`: 提交申请成功后的反馈状态。
- **约束**:
  - 状态切换不能依赖作品标题字段。
  - 去除作品标题字段后，用户仍能区分描述区、提示词区和操作区。
