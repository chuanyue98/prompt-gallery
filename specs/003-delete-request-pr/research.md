# 研究报告：详情页删除请求流程

## 决策：删除实现机制

### 方案：通过 GitHub Git Data API 手动重建树
- **具体选择**：使用 `octokit.git.getTree` (recursive: true) 获取指定目录下的所有文件，然后通过 `octokit.git.createTree` 将这些文件的 `sha` 设为 `null` 来实现逻辑删除。
- **技术理由**：GitHub REST API 没有直接删除目录的端点。手动重建树可以在一个 Commit 内删除整个文件夹及其内容，且无需引入额外的插件（如 `octokit-plugin-create-pull-request`），保持依赖轻量。

## 决策：UI 表现

### 方案：幽灵按钮 (Ghost Button)
- **具体选择**：在 Modal 底部或侧边栏增加一个 `text-slate-500` 的超链接或幽灵按钮，仅在悬停时略微加深颜色。
- **技术理由**：满足用户“不要太大太显眼”的需求，避免干扰核心的浏览和复制体验。

## 考虑过的替代方案

### 方案 A：直接在 main 分支删除
- **评估**：风险过高，且违反了项目“通过 PR 提交”的原则。
- **结论**：拒绝。

### 方案 B：使用单独的删除申请表单
- **评估**：增加了用户的操作负担。
- **结论**：拒绝，直接在详情页触发简单确认即可。
