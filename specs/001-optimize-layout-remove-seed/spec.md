# 功能规格说明书：优化画廊布局并移除 Seed 参数

**功能分支**: `001-optimize-layout-remove-seed`  
**创建日期**: 2026-04-18  
**状态**: 草案  
**输入**: 用户描述: "去除seed 参数 还有 页面上可显示的 item多一点。现在一行只有三个有些浪费"

## 用户场景与测试 *(必填)*

### 用户故事 1 - 更紧凑的画廊布局 (优先级: P1)

作为 Prompt Gallery 的访客，我希望在画廊页面每行看到更多的项目，以便我能更高效地浏览集合，而无需过多的滚动。

**为什么选择此优先级**: 直接回应了用户关于空间“浪费”的反馈，并提升了核心浏览体验。

**独立测试**: 可以通过打开画廊页面并验证在标准桌面屏幕上每行的项目数量是否大于 3 来进行测试。

**验收场景**:

1. **假设** 使用桌面屏幕（宽度 > 1280px）, **当** 我查看画廊时, **那么** 我应该看到每行至少有 4 或 5 个项目，而不是 3 个。
2. **假设** 使用平板电脑屏幕, **当** 我查看画廊时, **那么** 网格应根据响应式调整为比当前布局更高的密度。

---

### 用户故事 2 - 简化参数显示 (优先级: P2)

作为查看特定提示词详情的用户，我希望从界面中移除 “Seed” 参数，以便 UI 保持简洁，并仅专注于最核心的元数据。

**为什么选择此优先级**: 根据用户移除较不相关的技术数据的要求，清理 UI。

**独立测试**: 可以通过检查画廊卡片和详情/模态框视图，确保看不到 “Seed” 字段来进行测试。

**验收场景**:

1. **假设** 查看画廊项目卡片, **当** 我查看显示的参数时, **那么** “Seed” 值不应存在。
2. **假设** 打开投稿或详情模态框, **当** 我查看提示词元数据时, **那么** “Seed” 字段应该消失。

---

## 需求 *(必填)*

### 功能需求

- **FR-001**: 必须更新画廊网格布局，以支持更高的项目密度（例如，大屏幕上为 4 或 5 列）。
- **FR-002**: 必须保持响应式设计，确保网格在移动端、平板电脑和桌面端断点之间优雅缩放。
- **FR-003**: 必须从所有公共 UI 组件（画廊卡片、模态框等）中移除 “Seed” 参数。
- **FR-004**: 如果内部逻辑不再需要，数据同步脚本（`scripts/sync.ts`）应当停止在生成的静态 JSON 中包含 `seed` 字段。
- **FR-005**: 投稿表单/模态框必须更新，移除 “Seed” 输入字段。

### 关键实体 *(如果功能涉及数据，请包含此部分)*

- **GalleryItem**: 代表一个提示词条目。其元数据模式将被更新，排除 `seed` 属性。

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 桌面端（>=1280px）的画廊显示密度从每行 3 个项目增加到至少每行 4 个项目。
- **SC-002**: 用户界面中 “Seed” 字段的可见性 100% 移除。
- **SC-003**: 移动端响应式无回归；画廊在窄至 320px 的屏幕上仍保持可用。

## 假设

- 从 UI 中移除 “Seed” 参数不需要立即从现有的 `index.md` 文件中删除 `seed` 字段（以保留数据），但它将不再显示或被要求填写。
- 用户更倾向于大屏桌面上的 4 或 5 列布局，以减少“空间浪费”。
