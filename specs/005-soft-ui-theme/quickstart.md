# Quickstart：Soft UI 主题切换

## 1. 准备

1. 安装依赖：`npm install`
2. 启动开发环境：`npm run dev`
3. 打开首页并确认当前默认主题仍为现有 cyber-obsidian 风格

## 2. 手动验证场景

### 场景 A：主题切换可发现且即时生效

1. 打开首页顶部导航
2. 找到顶部导航中的 `THEME` 主题切换选项框
3. 从默认的 `暗曜` 主题切换到 `Soft UI`
4. 确认页面背景、导航、搜索框、筛选器、卡片和按钮立即变为柔和浮雕风格
5. 切回 `暗曜` 主题，确认界面恢复当前暗色玻璃风格且无 Soft UI 残留
6. 检查导航中的图形标识与 `PROMPT GALLERY` 标题在两种主题下都与背景协调，不显得突兀

### 场景 B：主题偏好持久化

1. 切换到 Soft UI
2. 刷新页面
3. 确认页面仍保持 Soft UI 主题
4. 关闭并重新打开页面
5. 再次确认导航中的主题选中态与页面主题一起被恢复

### 场景 C：画廊高频操作在 Soft UI 下正常工作

1. 在 Soft UI 主题下输入搜索词
2. 切换“全部 / 视频 / 图片”分类
3. 打开作品详情弹层
4. 使用复制按钮和来源按钮
5. 确认所有控件状态清晰，hover/active 有上浮或下沉反馈

### 场景 D：投稿弹层的 Soft UI 控件一致性

1. 在 Soft UI 主题下打开投稿弹层
2. 在“上传文件 / Media URL”模式间切换
3. 检查输入框、按钮、预览区、提交按钮的圆角、阴影和聚焦样式是否一致
4. 确认主要操作按钮仍是视觉焦点，文字和提示文案清晰可读

### 场景 E：回退与边缘情况

1. 快速来回切换主题多次
2. 检查页面是否出现局部残留样式或混合状态
3. 检查在移动端宽度下导航与主题切换入口是否仍可用
4. 检查未专门美化的文本区域是否至少保持足够对比度

## 3. 自动验证

1. 运行 `npm run lint`
2. 运行 `npm run test`
3. 必要时运行 `npm run build`

## 4. 完成标准

- 主题切换入口可见、可操作
- Soft UI 主题与默认主题切换即时且持久化
- 关键界面遵循柔和浮雕视觉语义
- 现有核心浏览、详情和投稿流程未回归
