# UI 合约：Soft UI 主题切换

## 1. 主题切换入口

- 主题切换入口必须在全局导航或等价的高可见位置出现。
- 控件必须采用统一的选项框/下拉选择模式，明确展示当前主题，并允许用户切换到至少“默认主题”和“Soft UI”两个选项。
- 该控件的结构必须允许后续继续扩展更多主题，而不需要改成另一种交互形态。
- 用户切换主题后，当前页面不得整页跳转或丢失上下文。

## 2. 主题应用范围

- 主题切换至少影响以下区域：
  - 应用背景与整体页面氛围。
  - 导航条与主题切换入口本身。
  - 画廊搜索框、分类切换器、卡片表面。
  - 详情弹层、复制按钮、来源按钮、申请下架输入区。
  - 投稿弹层中的预览区、模式切换器、输入框、提交按钮。
- 尚未专门强化的区域也必须继承基础 token，确保可读性和可操作性。
- 导航中的品牌标题与图形标识也必须纳入同一视觉协调范围，避免与背景、主视觉或主题风格冲突。

## 3. Soft UI 视觉语义

- 背景：采用浅色或低饱和渐变，避免高对比硬切分区。
- 表面：卡片、按钮、输入容器使用大圆角与柔和材质感。
- 阴影：通过低透明度亮/暗双阴影体现浮雕关系，不使用强烈硬边投影。
- 强调：主操作允许使用少量强调色，但不能破坏整体柔和氛围。
- 图标：保持线性、克制，不引入过度装饰性图形。

## 4. 交互状态

- Hover: 表面阴影加深或轻微上浮，让用户感知可交互性。
- Active: 表面阴影收敛或反转，形成按压下沉效果。
- Focus: 必须有清晰且不刺眼的聚焦表现。
- Selected: 当前主题选项与当前分类/模式按钮应具有稳定的选中态区分。
- Disabled: 禁用态需保持可辨识，但不能误导用户为可点击。

## 5. 持久化行为

- 用户选择任一主题后，刷新页面或重新进入应用时必须恢复该主题。
- 当本地偏好不可用或值无效时，系统必须回退到默认主题。
- 切回默认主题后，不得残留 Soft UI 表面或阴影样式。

## 6. 验证要点

- 切换到 Soft UI 后，关键界面表面应明显从 cyber-obsidian 深色玻璃风格转为柔和浮雕风格。
- 切换前后，用户仍能完成搜索、筛选、查看详情、复制内容和投稿等主要任务。
- 长时间浏览时，文字、图标、标签和主要按钮层级依然清晰。
