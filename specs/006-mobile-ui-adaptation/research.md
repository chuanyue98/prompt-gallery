# 技术研究：移动端适配优化 (Mobile UI Adaptation)

## 研究综述

通过对 `Navbar`, `Gallery`, `ContributeModal` 及 `globals.css` 的代码审查，识别出当前布局在小屏幕（< 640px）下的核心失效点。本项目采用 TailwindCSS v4，将充分利用其移动优先（Mobile-First）的类体系进行重构。

## 关键决策

### 1. 导航栏 (Navbar) 精简策略
- **决策**: 在屏幕宽度 < 640px 时，将主题切换器从“文字标签 + 箭头”模式切换为“仅图标”或“极简文字”模式。
- **理由**: 目前的 `min-w-[11.5rem]` 会导致 Logo 和投稿按钮在 375px 宽度下发生挤压或溢出。
- **替代方案**: 隐藏主题切换器（不推荐，影响功能发现）。

### 2. 详情弹窗 (Detail Modal) 移动端重构
- **决策**: 
  - 在移动端移除 `rounded-[3rem]`，改用 `rounded-t-[2rem]` 并实现全屏覆盖（Full Screen Overlay）。
  - 将桌面端的左右布局（左媒体，右内容）调整为上下垂直布局。
  - 减小 Overlay 的内边距（从 `p-6` 减小为 `p-0` 或 `p-4`）。
- **理由**: 窄屏下左右布局会导致内容极度拥挤，大圆角在全屏下视觉效果不佳且浪费空间。

### 3. 画廊网格 (Gallery Grid) 密度调整
- **决策**: 移动端默认使用 `grid-cols-2`（每行 2 列），但在极窄屏幕（< 340px）下回退到 `grid-cols-1`。
- **理由**: 目前的 `grid-cols-1` 使卡片过大，导致首屏内容量过低。
- **替代方案**: 始终保持 `grid-cols-1`（会导致滚动距离过长）。

### 4. 投稿表单 (Contribute Modal) 交互优化
- **决策**: 
  - 移除媒体预览区的固定高度 `min-h-[400px]`，改为自适应或比例限制。
  - 将表单中的 2 列布局（Model & Tags）在移动端降级为单列。
- **理由**: 减少垂直空间的无效占用，确保输入法弹出时不影响核心表单操作。

## 最佳实践参考
- **Tailwind v4 容器**: 优先使用百分比宽度和 `max-w-full`。
- **Soft UI 适配**: 在小屏幕上减少阴影的扩散半径（Spread），防止视觉上的“脏感”。
- **触控区域**: 确保所有按钮高度不低于 `44px`。
